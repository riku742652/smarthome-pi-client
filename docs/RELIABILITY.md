# Reliability Requirements - Smarthome Pi Client

## パフォーマンス要件

| 指標 | 目標値 |
|------|--------|
| スキャン間隔 | 60秒（設定可能） |
| 1回のスキャン時間 | 5秒（設定可能） |
| POST タイムアウト | 10秒 |
| システム再起動時の自動復旧 | systemd の `Restart=on-failure` で対応 |

## 可用性要件

- Pi が再起動後に自動的にサービスが再開すること
- ネットワーク障害から回復後に送信が再開すること

## エラーハンドリング

- HTTP エラー: ログに記録し、次のサイクルで再試行
- BLE スキャン失敗: 警告ログを記録し、次のサイクルで再試行
- 予期しない例外: エラーログを記録し、次のサイクルで継続
